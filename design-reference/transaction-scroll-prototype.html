Scroll-Driven Animations · Requires Chrome 115+ · Safari 17.2+ · Firefox 110+
Fallback: IntersectionObserver triggers on older browsers (auto-detected, no JS framework needed)

How it works

CSS Scroll-Driven Animation (the magic)

Each card has animation-timeline: view(block) which ties the animation progress directly to the card's position in the scroll container — no JS, no ResizeObserver. animation-range: entry 0% entry 38% plays the animation only during the "entering" phase (card crossing the top of the viewport).

SwiftUI equivalent

This replicates .scrollTransition { content, phase in content.opacity(...).scaleEffect(...).offset(...) } exactly. The phase.isIdentity check maps to the element being fully in view (animation at 100%). Entry → animated. Identity → normal.

The keyframe

Cards glide in from translateX(-28px) scale(0.88) blur(3px) to translateX(0) scale(1) blur(0) — matching Subbie's slide + fade effect. The will-change: transform, opacity forces GPU compositing for 60fps.

IntersectionObserver fallback

The @supports not (animation-timeline: view()) block applies transition-based animation triggered by the JS observer below — same visual result, broader browser support.
